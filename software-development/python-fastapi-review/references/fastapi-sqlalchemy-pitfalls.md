# FastAPI + SQLAlchemy Model Anti-Patterns

Common structural issues found in student/team FastAPI projects.

## 1. Column() Chinese-as-name Trap

```python
# ❌ BAD: Chinese string as first positional arg → database column IS Chinese
id = Column("学生编号", Integer, primary_key=True)
name = Column("姓名", String(50))

# ✅ GOOD: Use comment= keyword for descriptions
id = Column(Integer, primary_key=True, autoincrement=True, comment="学生编号")
name = Column(String(50), nullable=False, comment="姓名")
```

Impact: All DB columns end up with Chinese names. Hard to query.
Find with: `grep -rn 'Column("' --include="*.py" | grep -v "comment="`

## 2. Missing primary_key=True

```python
# ❌ unique=True alone is NOT a primary key
class_id = Column(String(20), unique=True, nullable=False, index=True)

# ✅
class_id = Column(String(20), primary_key=True, comment="班级编号")
```

Impact: `ArgumentError: could not assemble any primary key columns`

## 3. Auto-increment id in Create Schema

```python
# Model: autoincrement=True
id = Column(Integer, primary_key=True, autoincrement=True)

# ❌ Schema: id is required in create form
class EmploymentCreate(BaseModel):
    id: int = Field(...)  # user must fill this → duplicate key errors

# ✅ Schema: omit auto-generated fields
class EmploymentCreate(BaseModel):
    class_id: str = Field(...)
    # id is NOT here — auto-generated by DB
```

Impact: `IntegrityError: Duplicate entry for key 'PRIMARY'`

## 4. Schema↔Model Type Mismatch

```python
# Model: Integer
id = Column(Integer, primary_key=True)

# Schema: str
class Response(BaseModel):
    id: str  # ❌ Model returns int → ResponseValidationError
```

Impact: Insert succeeds but response serialization fails with 500.
Fix: Match schema type to model column type exactly.

## 5. Duplicate Model Definitions

```python
# app/employment.py
class Employment(Base):
    __tablename__ = "employment"
    class_id = Column(...)

# app/models/employment.py
class Employment(Base):
    __tablename__ = "employment"  # ← Same name, different fields!
    class_no = Column(...)
```

Impact: `InvalidRequestError: Table 'employment' is already defined`
Fix: Delete the unused file. One table = one model definition.
Find with: `grep -rh "__tablename__" --include="*.py" | sort | uniq -d`

## 6. Column Without Type (NullType)

```python
# ❌ no type → NullType
teacher_id = Column("教师编号", primary_key=True, autoincrement=True)

# ✅
teacher_id = Column(Integer, primary_key=True, autoincrement=True, comment="教师编号")
```

Impact: `CompileError: Can't generate DDL for NullType()`

## 7. Multiple Entry Points

```python
# root/main.py  → uses common/ modules
app = FastAPI(...)

# app/main.py   → uses app/ modules  
app = FastAPI(...)
```

Two different app instances with different configs, databases, and route registrations. Only one should exist. Delete the stale one.

## 8. Table Name is SQL Reserved Word

```python
__tablename__ = "class"  # class is a MySQL keyword
```

Use backticks in raw SQL or rename to `classes`.
