# Technical Tutorial Authoring (技术教程编写)

## Overview

当用户要求"把XX内容整理成教程/MD文件"、"根据网页+自己补充写一份XX教程"时，使用此技能。
核心流程：浏览器提取受限网页内容 → 结合自身知识补充 → 结构化输出 MD 文件。

## When to Use

- 用户提供网页链接，要求提取内容并整理成教程
- 用户要求"把XX知识写成markdown教程"
- 需要从飞书/语雀/Notion等受限文档页面提取信息后补充完整
- 用户在学习某技术（如Docker、Git、MySQL），需要系统化的学习文档
- **注意**：本技能负责内容整理和文档编写，不负责代码审查或项目分析

## 工作流程

### Step 1: 提取网页内容
使用 `browser_navigate` 访问链接，然后：
- 通过 `browser_snapshot(full=true)` 获取页面文字内容
- 点击目录中的各个章节，逐一提取内容
- 注意页面可能的**登录遮罩**——未登录状态下只能读取部分内容，记下能读到的内容

### Step 2: 整理网页原始信息
将提取到的文字归纳：
- 列出文档的**目录结构**（各章节标题）
- 记录每个章节的**关键语句**（原文要点）
- 标注哪些是**图片被挡**或**内容不可见**的部分

### Step 3: 结合自身知识补充
对于网页中缺失/不完整的内容，用自己的知识补充：
- 常见命令的完整表格（命令+说明+示例）
- 核心概念的类比解释（如"容器=集装箱"）
- 实用技巧和注意事项（如数据卷挂载的坑）
- 速查表/附录（方便日后翻阅）

### Step 4: 输出结构化 MD 文件
文件结构模板：

```markdown
# XX 教程

## 目录
1. [XXX](#1-xxx)

## 1. XXX
### 1.1 概念
### 1.2 用法

## 常见命令
| 命令 | 说明 | 示例 |
|------|------|------|

## 附录：速查表
```

### Step 5: 保存到桌面或用户指定位置
使用 `write_file` 写入文件，路径优先选 `~/Desktop/` 或用户指定路径。

### 提示信息
> 💡 **提示**：补充性说明
> ⚠️ **注意**：需要警惕的坑
> ❌ **常见错误**：容易犯的错

## 用户偏好
- 用户偏好**表格对比**（如绑定挂载 vs 命名卷的对比表）
- 用户偏好**类比/人话解释**（如"容器就是集装箱"）
- 用户偏好**分点说明**，而不是大段文字
- 教程尾部附**速查表**方便日后翻阅

## Common Pitfalls

1. **飞书文档登录遮挡**：未登录状态下图片无法访问，不要硬等——直接用自身知识补充
2. **页面 ref 失效**：页面导航后 ref 可能改变，需要重新 `browser_navigate` 获取新 ref
3. **内容太冗长**：单篇 MD 控制在 15-20KB 以内，过长的内容拆到 `references/` 子文件中
4. **遗漏实操命令**：教程必须附带可以直接运行的命令示例，不能只有概念描述
5. **忽略版本差异**：Docker 等工具版本可能变化，标注版本信息（如 `mysql:8.0`）

## Verification Checklist

- [ ] 网页内容已全部提取并整理
- [ ] 缺失部分已用自身知识合理补充
- [ ] 有表格对比关键概念
- [ ] 每个命令都可直接复制运行
- [ ] 有速查表或附录
- [ ] 文件保存到用户期望的路径
- [ ] 文件编码为 UTF-8，中文显示正常
